## 一、任务目标

在 compose.html 的 Iframe 沙箱中集成富文本编辑器,为用户提供基本的邮件撰写和格式化能力。

---

## 二、技术方案

### 2.1 编辑器选型

**选择 Quill 编辑器**,理由:
- 轻量级 (~43KB gzipped)
- API 简洁,易于集成
- 支持模块化配置
- 可通过 CDN 引入,无需复杂构建
- 文档完善,社区活跃

**备选方案**: TinyMCE (功能更强大但体积较大)

### 2.2 集成方式

1. **通过 CDN 引入 Quill**
   - 在 compose.html 中添加 Quill CSS 和 JS
   - 版本: 使用最新稳定版 (2.0.x)

2. **创建编辑器封装模块**
   - 文件: `src/compose/editor.ts`
   - 职责: 初始化编辑器、配置工具栏、提供内容读写接口

3. **工具栏配置**
   - 基础格式: 粗体、斜体、下划线、删除线
   - 标题: H1, H2, H3
   - 列表: 有序列表、无序列表
   - 对齐: 左对齐、居中、右对齐
   - 链接: 插入超链接
   - 颜色: 文字颜色、背景色

### 2.3 接口设计

```typescript
// src/compose/editor.ts
export class ComposeEditor {
  private quill: Quill;

  constructor(container: HTMLElement, options?: EditorOptions);

  // 获取编辑器内容
  getHTML(): string;
  getText(): string;

  // 设置编辑器内容
  setHTML(html: string): void;
  setText(text: string): void;

  // 清空编辑器
  clear(): void;

  // 获取纯文本长度
  getLength(): number;

  // 编辑器状态
  isEmpty(): boolean;

  // 事件监听
  onChange(callback: (content: string) => void): void;
}
```

### Step 2: 创建 editor.ts 模块

```typescript
// src/compose/editor.ts
import Quill from 'quill'; // 类型定义

export interface EditorOptions {
  placeholder?: string;
  theme?: 'snow' | 'bubble';
  modules?: any;
}

export class ComposeEditor {
  private quill: Quill;

  constructor(container: HTMLElement, options: EditorOptions = {}) {
    const defaultOptions = {
      theme: 'snow',
      placeholder: '撰写邮件内容...',
      modules: {
        toolbar: [
          ['bold', 'italic', 'underline', 'strike'],
          [{ 'header': [1, 2, 3, false] }],
          [{ 'list': 'ordered'}, { 'list': 'bullet' }],
          [{ 'align': [] }],
          ['link'],
          [{ 'color': [] }, { 'background': [] }]
        ]
      }
    };

    this.quill = new (window as any).Quill(container, {
      ...defaultOptions,
      ...options
    });
  }

  getHTML(): string {
    return this.quill.root.innerHTML;
  }

  getText(): string {
    return this.quill.getText();
  }

  setHTML(html: string): void {
    this.quill.root.innerHTML = html;
  }

  setText(text: string): void {
    this.quill.setText(text);
  }

  clear(): void {
    this.quill.setText('');
  }

  getLength(): number {
    return this.quill.getLength();
  }

  isEmpty(): boolean {
    return this.quill.getText().trim().length === 0;
  }

  onChange(callback: (content: string) => void): void {
    this.quill.on('text-change', () => {
      callback(this.getHTML());
    });
  }
}
```

### Step 3: 在 compose.ts 中集成

```typescript
// src/compose.ts
import { ComposeEditor } from './compose/editor';

let editor: ComposeEditor;

function initEditor() {
  const container = document.getElementById('editor-container');
  if (!container) {
    console.error('Editor container not found');
    return;
  }

  editor = new ComposeEditor(container, {
    placeholder: '撰写邮件内容...'
  });

  // 监听内容变化
  editor.onChange((content) => {
    console.log('Content changed:', content.substring(0, 50));
  });
}

// 页面加载完成后初始化
window.addEventListener('DOMContentLoaded', () => {
  initEditor();
});
```

## 六、风险与应对

### 风险 1: CDN 加载失败
**影响**: 编辑器无法初始化
**应对**:
- 添加加载超时检测
- 显示友好的错误提示
- 考虑后续版本本地化 Quill 资源

### 风险 2: CSP 策略限制
**影响**: CDN 资源被浏览器拦截
**应对**:
- 在 manifest.json 中正确配置 CSP
- 测试各个邮箱站点的 CSP 兼容性

### 风险 3: Iframe 中的样式冲突
**影响**: 编辑器显示异常
**应对**:
- 使用 Shadow DOM 隔离样式(如果需要)
- 测试并调整 CSS 优先级
