## 四、人工验收测试用例

### TC-01: 富文本编辑器基础功能

**测试目标**: 验证编辑器能正常工作并支持基本格式化

**测试步骤**:
1. 打开 Gmail 页面
2. 点击 FAB 按钮打开写信面板
3. 在编辑器中输入文本
4. 尝试使用工具栏的格式化功能:
   - 粗体、斜体、下划线
   - 标题(H1, H2, H3)
   - 有序列表、无序列表
   - 文字颜色
5. 截图编辑器状态

**验收标准**:
- [ ] 编辑器正常显示
- [ ] 能输入和编辑文本
- [ ] 所有格式化按钮正常工作
- [ ] 格式化效果正确显示

---

### TC-02: 收件人管理功能

**测试目标**: 验证收件人添加、验证和删除功能

**测试步骤**:
1. 在收件人输入框中输入有效邮箱地址(如 `test@example.com`)
2. 点击"添加"按钮
3. 尝试添加无效邮箱地址(如 `invalid-email`)
4. 尝试添加重复的邮箱地址
5. 添加多个有效收件人
6. 点击删除按钮删除一个收件人
7. 截图收件人列表状态

**验收标准**:
- [ ] 有效邮箱能成功添加
- [ ] 无效邮箱被拒绝并显示错误提示
- [ ] 重复邮箱被拒绝并提示
- [ ] 收件人列表正确显示
- [ ] 收件人计数正确
- [ ] 删除功能正常工作

---

### TC-03: 主题输入功能

**测试目标**: 验证主题输入框正常工作

**测试步骤**:
1. 在主题输入框中输入邮件主题
2. 输入包含特殊字符的主题
3. 输入很长的主题(超过 100 字符)
4. 截图主题输入状态

**验收标准**:
- [ ] 主题输入框正常显示
- [ ] 能输入和编辑主题
- [ ] 支持特殊字符
- [ ] 长主题能正常显示

---

### TC-04: 完整发送流程

**测试目标**: 验证从撰写到发送的完整流程

**测试步骤**:
1. 添加至少一个收件人
2. 输入邮件主题
3. 在编辑器中输入邮件内容(包含格式化)
4. 点击"发送"按钮
5. 观察发送状态提示
6. 打开 DevTools Console 查看日志
7. 截图发送过程和 Console 日志

**验收标准**:
- [ ] 点击发送按钮触发发送流程
- [ ] 显示"正在发送"状态
- [ ] Console 中有加密日志
- [ ] Console 中有 postMessage 发送日志
- [ ] 显示"发送成功"提示
- [ ] 发送成功后表单自动清空

---

### TC-05: 表单验证

**测试目标**: 验证各种异常情况的处理

**测试步骤**:
1. 不添加收件人,直接点击发送
2. 添加收件人但不输入主题,点击发送
3. 添加收件人和主题但不输入内容,点击发送
4. 截图每种情况的错误提示

**验收标准**:
- [ ] 无收件人时显示错误提示
- [ ] 无主题时显示错误提示
- [ ] 无内容时显示错误提示
- [ ] 所有错误提示友好明确

---

### TC-06: 取消功能

**测试目标**: 验证取消按钮能关闭面板

**测试步骤**:
1. 打开写信面板
2. 输入一些内容
3. 点击"取消"按钮
4. 观察面板是否关闭

**验收标准**:
- [ ] 点击取消按钮关闭面板
- [ ] 面板以侧滑动画关闭
- [ ] Console 中有 CLOSE_PANEL 消息日志

---

### TC-07: 加密功能验证

**测试目标**: 验证加密逻辑正确执行

**测试步骤**:
1. 在 DevTools Console 中执行测试函数:
   ```javascript
   window.testEncryption()
   ```
2. 查看 Console 输出
3. 截图加密测试结果

**验收标准**:
- [ ] 加密测试成功执行
- [ ] 生成的加密包格式正确
- [ ] 加密包包含所有必需字段
- [ ] 序列化和反序列化正常

---

### TC-08: postMessage 通信验证

**测试目标**: 验证 Iframe 与宿主的通信

**测试步骤**:
1. 完成一次完整的发送流程
2. 在 DevTools Console (top context) 中查看日志
3. 确认收到 SEND_EMAIL 消息
4. 确认发送了 SEND_EMAIL_RESPONSE 响应
5. 截图 Console 日志

**验收标准**:
- [ ] 宿主页面收到 SEND_EMAIL 消息
- [ ] 消息包含正确的 payload 结构
- [ ] Iframe 收到 SEND_EMAIL_RESPONSE 响应
- [ ] 响应包含 success 和 messageId

---

### TC-09: 多邮箱兼容性测试

**测试目标**: 验证在不同邮箱站点的兼容性

**测试步骤**:
1. 依次在以下站点测试完整发送流程:
   - Gmail (https://mail.google.com)
   - QQ 邮箱 (https://mail.qq.com)
   - 163 邮箱 (https://mail.163.com)
   - Outlook (https://outlook.live.com)
2. 每个站点都截图写信面板

**验收标准**:
- [ ] 所有站点的写信面板正常显示
- [ ] 所有站点的功能正常工作
- [ ] 样式在所有站点一致
- [ ] 无明显的兼容性问题

---

### TC-10: UI/UX 体验测试

**测试目标**: 验证用户体验是否流畅

**验收标准**:
- [ ] 编辑器输入流畅无卡顿
- [ ] 收件人操作响应及时
- [ ] 发送状态反馈清晰
- [ ] 错误提示友好易懂
- [ ] 整体 UI 美观协调
